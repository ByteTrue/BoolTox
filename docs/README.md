# BoolTox 文档索引

> 「好品味的文档结构 = 一眼就能找到需要的内容。」

## 产品线（Product）

- `product/performance/agent-performance.md`：Agent 性能剖析与测试结论。
- `product/performance/ui-optimization-summary.md`：Web 端性能优化方案及指标。
- `product/web/README.md`：Web 端整体设计/发展记录。
- `product/web/quick-start.md`：Web 团队快速启动与环境说明。
- `product/web/integration-guide.md`：与其它系统对接的 API/集成指南。
- `product/web/migration-summary.md`：历史版本迁移/重构记录。
- `product/web/fixes-applied.md`：针对线上问题的补丁记录。
- `product/web/new-ui/*.md`：新 UI 研发的设计、施工、总结与指南。

## 插件生态（Plugins）

- `plugins/development-guide.md`：插件开发全流程与最佳实践。
- `plugins/management-redesign.md`：插件管理能力的重构方案。
- `plugins/repo-migration.md`：插件仓库迁移步骤与复盘。

## 历史与背景（History）

- `history/project-timeline.md`：项目演进时间线与关键里程碑。
- `history/wondrous-sniffing-otter.md`：重要会议/讨论纪要（归档）。

---

若要新增文档：先确认它属于哪一层（Product/Plugins/History），必要时再创建子目录，别再把文件随手塞到根目录。保持层级简洁，拒绝文档垃圾场。***
