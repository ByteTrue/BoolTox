# 设置页面使用指南

## 概述
新增的设置页面为用户提供了应用信息查看、版本更新检查等核心功能，界面简洁优雅，便于测试和管理应用。

## 功能模块

### 1. 应用信息
显示应用的基本信息：
- **应用名称**: Booltox 不二工具箱
- **当前版本**: 显示当前应用版本号
- **更新渠道**: stable/beta/alpha
- **API 地址**: 后台 API 服务地址

### 2. 开发者信息
显示开发相关信息：
- **环境**: 开发/生产环境
- **Electron**: 启用状态
- **Node 集成**: 启用状态

### 3. 版本更新
**功能:**
- 显示当前更新状态
- 手动检查更新按钮
- 更新详情展示（版本号、更新说明、文件大小）

**使用方式:**
1. 进入设置页面
2. 找到"版本更新"卡片
3. 点击右侧"检查更新"按钮
4. 等待检查结果

**状态说明:**
- ✅ **当前版本已是最新**: 无需更新
- 🔄 **正在检查更新**: 检查进行中
- 📦 **发现新版本**: 有可用更新
- ⬇️ **正在下载更新包**: 下载进行中
- ✓ **更新包已就绪**: 可以安装
- ❌ **检查失败**: 出现错误

3. 点击进入设置页面

### 快捷键（未来可扩展）

## 测试更新功能

### 方法一: 修改版本号测试
1. 打开 `src/renderer/config/app-info.ts`
2. 修改 `APP_VERSION` 为较低版本（如 `0.0.0`）
3. 重新构建: `npm run dev` 或 `npm run build`
4. 进入设置页面
5. 点击"检查更新"按钮
6. 应该会提示有新版本可用

### 方法二: 后台配置测试版本
1. 在后台系统创建新的发布版本
2. 设置版本号高于当前版本
3. 上传安装包和配置信息
4. 客户端设置页面检查更新
5. 应该能检测到新版本

## 常见问题

### Q: 点击"检查更新"没反应？
A: 
1. 检查网络连接
2. 检查后台 API 是否正常
3. 查看浏览器控制台是否有错误信息

### Q: 如何触发更新提示？
A: 
1. 修改客户端版本号为较低版本
2. 或在后台创建更高版本的发布
3. 点击"检查更新"按钮

### Q: 设置页面在哪里？
A: 左侧导航栏，图标为心跳波形 📈

## 技术实现

### 组件结构
```
SettingsPanel (设置页面主组件)
├── 两列布局
│   ├── SettingCard (应用信息)
│   └── SettingCard (开发者信息)
└── 全宽布局
    └── SettingCard (版本更新)
```

### 响应式设计
- 桌面端: 2列网格布局 (`grid-cols-2`)
- 移动端: 1列堆叠布局 (`grid-cols-1`)
- 最大宽度: `max-w-4xl` (居中显示)

### 样式优化
- 卡片圆角: `rounded-2xl` (更紧凑)
- 内边距: `p-5` (适中)
- 标题图标: `size={18}` (适中)
- 文字大小: `text-xs` ~ `text-base` (紧凑)
