# 更新说明示例

## 🎉 新功能

- 新增自动更新功能，支持从 GitHub/GitLab Release 自动下载安装包
- 新增更新详情弹窗，展示完整 Release Notes
- 新增更新进度条，实时显示下载进度

## 🐛 修复

- 修复设置面板版本号显示为 0.0.0 的问题
- 修复更新检查无限转圈的问题
- 修复"稍后提醒"按钮清空更新信息的 bug

## ⚡ 优化

- 优化更新 UI 交互，状态切换更流畅
- 优化发布脚本，支持交互式 TUI 和 Markdown 更新说明
- 优化玻璃拟态效果，适配深色/浅色主题

## 📝 使用说明

将此文件放在 `resources/announcements/` 目录下，命名为 `v{版本号}.md`（如 `v0.0.2.md`）。

在运行 `pnpm release:cli` 发布时：

1. 可以在配置中指定默认的更新说明文件路径
2. 也可以在每次发布时临时指定一个文件路径
3. 或者直接粘贴 Markdown 文本内容

发布后，该内容会自动作为 GitHub/GitLab Release 的描述（Release Notes）。
